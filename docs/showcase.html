01 — the agent kernel

One gate, where the tool call is the checkpoint

An in-process kernel sits between an agent and the tools it calls. Everything the agent does to the outside world becomes a syscall through a boundary the model can't route around.

fak the agent kernel

“Treat the model like an untrusted program, and the tool call like a syscall.”

Apache-2.0 Go 1.26+ one static binary offline-capable

Everything an agent does to the outside world — call a tool, admit a result into memory, reuse a cached answer — becomes a syscall that passes through a kernel the model doesn't control. From the security seat it's a permission gate the agent can't argue past. From the performance seat it's a way to do shared work once instead of every turn. The surprise: those turn out to be the same gate.

flip 1 Policy in the kernel

The permission check rides the same call path as the tool call — one address space, no IPC, default-deny. Refusing an irreversible action doesn't depend on catching an attack; the lever was simply never wired up. When the check would crash or time out, the call doesn't run anyway — it fails closed.

flip 2 Addressable KV cache

Reach into the middle of a kept run, cut one span — a poisoned tool result, an expired secret — and leave the model's scratchpad bit-for-bit identical to a run that never saw it (max|Δ| = 0). The cache stops being just a speed structure and becomes one policy can address.

⚖ we keep score — what's real, what's not
  • The permission gate, quarantine, local-answer shortcut, in-kernel model (math proven exact against a reference), and the OpenAI-compatible gateway are shipped and on the critical path, each closed by a test.
  • A 29-claim prior-art audit scored 0/29 novel — every primitive is established; the contribution is the assembly: one in-process gate where the tool call is the checkpoint.
  • Power/energy numbers are simulated. The reuse win is self-host only, and frontier-scale fleet numbers are design targets, not measurements.

Built on — and kept honest by — DOS

This repo dogfoods DOS (dos-kernel), a separate trust substrate for fleets of agents: a claim is believed only when an independent witness — git ancestry, a file read-back, a real artifact — says so, never because a worker reported it done. Its dos.toml declares fak's real lane partition and turns the project's architectural invariants into a closed, verifiable refusal vocabulary (OFF_TRUNK, PUBLIC_LEAK). Different scope, same idea: don't take the agent's word for it — at the tool call, and at the “done.”
