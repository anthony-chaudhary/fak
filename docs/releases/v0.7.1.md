**TL;DR** ‚Äî Maintenance release that closes TICKETS **T4**: the two parallel OpenAI HTTP
seams are collapsed to one. The dead `engine.HTTPEngine` is removed and the
single-client topology is now an architecture invariant the build enforces, so it can't
re-entrench the way it did before. Also folds in the already-landed WSL test-runner
ergonomics. No served-path behavior change.

## One OpenAI client ‚Äî delete the dead HTTPEngine seam (#engine)

- **What changed** ‚Äî `engine.HTTPEngine` (+ `NewHTTPEngine`, its file-local
  `parseUsage`/`hashHex`, and its three tests) is deleted; `internal/engine` no longer
  imports `net/http`/`bytes`/`io`/`time`.
  - *Why:* it was a degenerate, never-wired second client ‚Äî it spoke a bespoke
    `tool=X args=Y` prompt instead of real tool-calling, registered only `mock`, and its
    sole caller was its own test. The seam had *entrenched* over time (the gateway added
    a third consumer of the live planner while this copy sat orphaned).
  - *How:* `agent.HTTPPlanner` already covered everything it offered and strictly more ‚Äî
    `base_url` local‚Üîremote swap (`adapter.Endpoint`), usage extraction, transcript
    witness, and a better retry (4-attempt 429/5xx backoff vs a 2-attempt transport-only
    loop) ‚Äî so the duplicate was removed, not folded. The Mock + cassette engines and the
    engine-residency adjudicator are unchanged.

## Build-enforced single-client invariant (#architest)

- **What changed** ‚Äî new attribute test `architest.TestSingleOpenAIChatClient`.
  - *Why:* T4 re-entrenched last time precisely because nothing stopped a second client
    from appearing; a convention is not a guarantee.
  - *How:* it asserts the OpenAI `chat/completions` endpoint appears ‚Äî as a string
    literal in non-test code ‚Äî in *exactly* `{agent (client), gateway (server route)}`.
    A new package referencing it (a re-grown duplicate) or either declared holder losing
    it fails the build. AST-based, so a doc comment mentioning the path doesn't count.

## Tooling ‚Äî WSL test ergonomics (#tooling)

- **What changed** ‚Äî `CLAUDE.md` documents the WSL **ext4-vs-9p** filesystem perf gap for
  the test hot path; `fak/test.ps1` / `fak/test.sh` gain a distro fallback (use
  `Ubuntu-24.04` if present, else WSL's default ‚Äî the old hardcoded distro broke nodes
  whose distro is just `Ubuntu`). *(Landed since v0.7.0; included here for completeness.)*
