**TL;DR** — New end-to-end Slack transfer commands make GPU server benchmark automation seamless. Session auditing accuracy restored after de-duplication bug was found and fixed.

> Current status: the historical `tools/bench_slack.py` bridge was sunset from the
> public tree; live Slack/GPU-server control now belongs in `fak-private`. See
> [`docs/gpu-server-private-boundary.md`](../gpu-server-private-boundary.md).

## Slack integration (#slack)

- **What changed** — New `transfer` and `download-and-register` commands in `bench_slack.py` automate the full GPU server benchmark lifecycle.
  - *Why:* Eliminate manual steps in GPU server benchmark workflow.
  - *How:* `tools/bench_slack.py:run_benchmark_and_transfer` + `tools/bench_slack.py:download_and_register`
  - Sends benchmark command via Slack control, polls for completion, downloads results, and registers in catalog automatically.
  - Fixed argparse name collision bug (`--command` → `--benchmark-command`).

## Tools (#tools)

- **What changed** — `session_audit.py` de-duplicates billed turns by `message.id` instead of counting each raw JSONL line.
  - *Why:* Turns-based billing was over-counting by ~2x due to counting split tool calls as separate turns.
  - *How:* `tools/session_audit.py` now groups by the root message.id and charges once per logical turn.
  - Test-locked with `session_audit_test.py` after fix confirmed.

## GPU experiments (#gpu)

- **What changed** — Reusable GPU/CPU Q8 crossover harness with VRAM guard.
  - *Why:* Make crossover experiments repeatable and prevent VRAM exhaustion.
  - *How:* `fak/experiments/gpu/gpu_crossover.ps1` + `fak/experiments/gpu/gpu_vram_guard.ps1`
  - Documents that 3B Q8 crossover FALSIFIES extrapolation on 8GB RX 7600 (test: `crossover-3b-rx7600-20260620`).

## GPU server

- **What changed** — GPU server benchmark catalog integration + python3 fix deployment.
  - *Why:* Track GPU server runs alongside other machines in the unified catalog.
  - *How:* `tools/bench_slack.py:register_gpu_server_run` + GPU server deployment summary.
  - Catalog now includes private GPU-server test run run entries from test runs.

## Memory (#memory)

- Cache-share-is-duration trap documented for session_audit users.
