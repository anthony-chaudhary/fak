Don't trust the agent.

Two shipped projects, built on one idea: treat the model like an untrusted program. Put the check on the call path — or in the git evidence — so the agent can't talk its way past it. Here's what that looks like, plus what's coming next.

01 — the agent kernel

fleet

The repository ships fak — an in-process kernel that sits between an agent and the tools it calls. One gate, where the tool call is the checkpoint.

github.com/anthony-chaudhary/fleet
fleet fak

“Treat the model like an untrusted program, and the tool call like a syscall.”

Apache-2.0 Go 1.26+ one static binary offline-capable

Everything an agent does to the outside world — call a tool, admit a result into memory, reuse a cached answer — becomes a syscall that passes through a kernel the model doesn't control. From the security seat it's a permission gate the agent can't argue past. From the performance seat it's a way to do shared work once instead of every turn. The surprise: those turn out to be the same gate.

flip 1 Policy in the kernel

The permission check rides the same call path as the tool call — one address space, no IPC, default-deny. Refusing an irreversible action doesn't depend on catching an attack; the lever was simply never wired up. When the check would crash or time out, the call doesn't run anyway — it fails closed.

flip 2 Addressable KV cache

Reach into the middle of a kept run, cut one span — a poisoned tool result, an expired secret — and leave the model's scratchpad bit-for-bit identical to a run that never saw it (max|Δ| = 0). The cache stops being just a speed structure and becomes one policy can address.

02 — the trust substrate

DOS

A domain-free trust substrate for fleets of autonomous agents. Where fak guards one agent's calls, DOS governs many agents working a shared repo at once.

github.com/anthony-chaudhary/dos-kernel · PyPI: dos-kernel
DOS dos-kernel

“The kernel is the part that doesn't believe the agents.”

MIT on PyPI Claude Code plugin git-evidence oracle

An agent will tell you it finished. DOS doesn't take its word for it. “Done” is confirmed against a witness the agent didn't author — git ancestry, a file read-back, a real artifact — not a line in a transcript. Refusals come from a closed, typed vocabulary you can verify, instead of free-text the model can negotiate with.

dos verify

Confirm a claim actually landed from git evidence, not a worker's self-report. A claimed “shipped” resolves to a real commit, or it doesn't.

dos arbitrate

Decide whether two workers can run concurrently without colliding on the same files — one lane lease per disjoint tree, so parallel sessions don't clobber each other.

dos refuse / check-reason

Refuse with a structured, verifiable reason from a closed set — OFF_TRUNK, PUBLIC_LEAK, ARCH_LAYER_VIOLATION — instead of an argument.

dos citation-resolve

Check that a cited legal case actually exists in a third-party reporter before anything relies on it — a concrete antidote to confident fabrication.

Witness-gated “done”

A harness goal or Stop-hook condition is normally checked by the model re-reading its own work — that's consistency, not grounding. DOS wires the Stop so it's refused until an independent witness corroborates the claimed effect. The self-stopping agent can't certify itself complete.

# the oracle binds a claim to real history — not to what the agent said dos verify fak architest # → SHIPPED ef239c9 (resolves to an actual commit) dos verify fak cachemeta # → NOT_SHIPPED (correct discrimination, with source)

This repo dogfoods DOS on itself: dos.toml declares fleet's real lane partition and turns the project's own architectural invariants into refusal vocabulary — the same discipline fak imposes on tool results, aimed at the fleet's own git workflow.

One idea, two seats

fak owns the boundary inside a single agent — the tool call is a syscall through a kernel the model can't control. DOS owns the boundary across a fleet — a claim is believed only when independent evidence says so. Different scope, same refusal to take the agent's word for it. That's the throughline, and it's the foundation everything here is built on.
